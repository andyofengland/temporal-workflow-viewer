# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the current default branch. We do not maintain separate branches for older releases unless otherwise stated.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it responsibly:

1. **Do not** open a public GitHub issue for security-sensitive bugs.
2. Open a **private security advisory** on GitHub: go to the repository → Security → Advisories → "Report a vulnerability", or contact the maintainers directly if you have a private channel.
3. Include a clear description, steps to reproduce, and impact if possible.
4. We will acknowledge your report and work with you to understand and address it.

We appreciate responsible disclosure and will credit reporters (with their permission) when the issue is fixed.
